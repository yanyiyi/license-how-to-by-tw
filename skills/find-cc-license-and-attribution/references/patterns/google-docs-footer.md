# Google Docs Footer Pattern

## Use When

Use this pattern for documentation pages where the license appears in a footer or site-wide statement such as `unless otherwise noted`, and the page may distinguish text, code samples, and trademarks.

## Main Risk

Do not collapse all page elements into one CC license. Documentation text may be CC, code samples may be under a software license, and logos or trademarks are usually excluded.

## Check

1. Find the exact footer or page statement.
2. Check whether it says `this page`, `this page and its contents`, `site content`, or another scope.
3. Check whether code samples have separate terms.
4. Check trademark or brand restrictions.
5. Confirm whether images, screenshots, or embedded media are covered or separately credited.

## Output Rows

| 使用對象 | Attribution / 應標示對象 | CC 授權名稱 |
|---|---|---|
| **文字內容／文件說明** | 文件頁面或發布機構指定標示 | 依 footer 或頁面授權正規化 |
| **程式碼範例** | 文件頁面或發布機構指定標示 | 非 CC：依頁面所列程式碼授權 |
| **Logo／商標／品牌特徵** | 各商標或標誌權利人 | 非 CC：不在授權範圍內 |

## Notes

If the footer uses `unless otherwise noted`, mention that exceptions still need separate checking.
